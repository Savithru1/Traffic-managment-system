Hyderabad Traffic Incident Management SystemA Java-based console application designed to log, manage, and prioritize traffic incidents across major intersections in Hyderabad. This project utilizes core Data Structures and Algorithms (DSA) to handle real-time data persistence and incident queuing.🚀 FeaturesPriority-Based Queue: Incidents are automatically sorted by severity ($P1$ to $P4$) using a custom Linked List implementation.Recent Activity Stack: Tracks the last 3 reported incidents using a bounded Stack (LIFO).Data Persistence: Automatically saves and loads incident data from a local .txt file.Zone Mapping: Pre-configured with 10 major Hyderabad zones (HITEC City, Gachibowli, Madhapur, etc.).Search & CRUD: Functionality to add, search by incident type, delete logs, and view total active counts.🛠️ Data Structures UsedStructureImplementationPurposePriority QueueSingly Linked ListTo ensure "Critical" (P1) incidents are addressed before "Low" (P4) ones.StackSingly Linked ListTo maintain a "Recently Viewed" history with a limit of 3 entries.File I/OBufferedReader / BufferedWriterFor persistent storage of logs on the local disk.💻 Setup & InstallationClone the repository:Bashgit clone https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
